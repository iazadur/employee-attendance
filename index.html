Overview

Project Overview

A web-based application that automates and digitizes employee attendance tracking, leave management, shift scheduling, and reporting for organizations.

Purpose

Replace manual paper-based attendance registers with an accurate, real-time digital system. Reduce administrative workload by 60–70%.

Goals

Provide transparent leave management, automated notifications, real-time dashboards, and role-based access control (Admin, Manager, Employee).

Requirements

Functional Requirements

  • Authentication & Authorization — JWT in HTTP-only cookies, role-based guards (Admin, Manager, Employee), session validation
  • Employee & Shift Management — CRUD for employees and shifts, activate/deactivate accounts, assign shifts
  • Attendance Tracking — Check-in/check-out, status tracking (present, late, half-day, absent, on-leave, holiday)
  • Leave Workflow — Submit leave requests, admin/manager review (approve/reject), leave history & balance tracking
  • Reports & Dashboard — Today KPIs, monthly attendance summaries, chart visualizations, analytics

Challenges

Challenges & Solutions

Problem JWT authentication across frontend and backend with cookie-based sessions and CORS configuration
Solution Implemented HTTP-only cookie strategy with Passport JS, proper CORS credentials, and session validation endpoint (/auth/me)
Problem Complex role-based access control across multiple user types with different permissions
Solution Built NestJS custom guards and decorators for RBAC, with role-aware frontend routing and conditional UI rendering
Problem Dockerized deployment with automated database migrations and environment configuration
Solution Created entrypoint script that runs prisma migrate deploy on startup. Externalized config via .env files and Docker Compose
